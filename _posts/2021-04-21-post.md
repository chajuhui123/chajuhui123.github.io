컴퓨터의 프로그램은 무엇으로 이루어져 있을까? **<프로그램 = 자료구조 + 알고리즘>** 이다. 대부분의 프로그램에서 자료를 처리하고 있고 이 자료는 **자료구조(data structure)**를 사용하여 저장된다. 또 주어진 문제를 해결하는 절차가 필요한데, 이를 **알고리즘(algorithm)**이라고 부른다. 

---

# 자료구조

프로그램에서 자료를 정리하여 보관하는 여러가지 구조들이 있는데, 이를 자료구조(data structure)라고 한다. 

### 자료구조의 목적

복잡한 자료들을 효율적으로 활용하기 위해서이다. 복잡한 자료를 빠르게 저장, 검색, 분석, 전송, 갱신하기 위해서는 자료구조가 효율적으로 조직화되어 있어야 한다. 각 응용에서 가장 적합한 자료구조와 알고리즘을 사용해야 한다.

### 자료구조의 분류

선형과 비선형으로 나눌 수 있다. 선형 자료는 데이터가 일렬로 나열되어 있는 것을 뜻하고, 비선형 자료구조는 특정한 형태를 띄고 있는 것을 뜻한다.

**선형 구조**

-배열

-연결리스트

-스택

-큐

**비선형 구조**

-트리

-그래프

# 알고리즘

알고리즘(algorithm)이란 어떠한 문제를 해결하기 위한 여러 동작들의 모임이다. 

### **알고리즘의 조건**

- **입력** : 외부에서 제공되는 자료가 0개 이상 존재한다.
- **출력** : 적어도 2개 이상의 서로 다른 결과를 내어야 한다.(즉 모든 입력에 하나의 출력이 나오면 안됨)
- **명확성** : 수행 과정은 명확하고 모호하지 않은 명령어로 구성되어야 한다.
- **유한성**(종결성) : 유한 번의 명령어를 수행 후(유한 시간 내)에 종료한다.
- **효율성** : 모든 과정은 명백하게 실행 가능(검증 가능)한 것이어야 한다.

### 알고리즘을 기술하는 방법

1. 자연어
2. 흐름도 (flow chart)
3. 의사 코드 (pseudo-code)
4. 프로그래밍 언어

이 중에서 **의사코드**와 **프로그래밍 언어**로 알고리즘을 기술하는 것이 가장 많이 쓰이는 방법이다. 프로그래밍 언어의 예약어들이 모두 명백한 의미를 가지고 있어 알고리즘을 작성하기에 적합하다. 의사 코드는 프로그래밍 언어보다 덜 엄격하고, 특정 언어에 의존적이지 않다.

### 알고리즘의 성능 분석

알고리즘 분석에서 2가지 측면을 고려할 수 있는데, 그것은 알고리즘의 수행시간과 알고리즘이 필요로 하는 기억 공간이다. 알고리즘의 수행시간 분석을 **시간복잡도(time complexticy)**라고 하고, 알고리즘이 사용하는 기억공간 분석을 **공간 복잡도(space complexity)**라고 한다.
